Page Features

Mobile-First Design

By default, this page is built for mobile devices with small displays. A look at the source code will shows that media query mixins (leveraging the PostCSS 'mixins' package) are used so that upon encountering a "small" (defined here as having a width of > 530px), medium (> 800px) or "large" (> 1010px) screen, the CSS fed to the browser will be modified in order to accomodate. Over 50% of global web traffic comes from mobile devices, and this figure is on the rise. Please feel free to view this page from your mobile device(s) and/or reshape your browser window to see how the page responds!

Minimal Data Usage

This page also features "responsive images", meaning that the server is hosting multiple versions of some images (e.g. the ocean-side cover image), and sending you the highest-quality version that your screen can display without sending one that contains un-displayable pixels. If you resize your browser, clear your cache, and 'inspect element' on the cover image, you will see that you are being sent the appropriate file each time. Additionally, the CSS and JS files hosted on the server are compressed and revisioned during the build process.

Blazing Fast Page Loads

In addition to responsive images, this page also utilizes a "lazy image loading" feature which means that when you first load the page, the server will only send your browser the image data which you are able to view immediately. As you scroll down the page and reach the other sections, your browser will then receive the image data necessary to display these sections. This allows for the user to only ever download data which they require. This was implemented with the Node.js 'Waypoints' and 'lazysizes' packages.

Clean, Reusable Code

Please take a peak at the source code for this page. As a matter of fact, feel free to conduct a thorough investigation! You will see that the CSS and JavaScript under-the-hood have been organized in a manner which allows for anyone to make a quick contribution, completely take over the reigns, or anything in-between. I've utilized the BEM methodology which allows for relatively easy long-term support. The CSS and JS are funneled through a number of gulp packages which allow for rapid development/testing and well-organized, lightweight source files being delivered to the end-user. The JS is modular, object-oriented, reusable, and I hope that you find it easily understandable.
